## 타입 추론

타입 추론이란 변수나 표현식의 타입을 명시적으로 선언하지 않아도 <br/>
컴파일러가 코드를 분석해 타입을 추론하는 것을 말합니다.

![bird](/images/posts/Typescript/type-assertion/bird.png)

위와 같이 타입스크립트는 bird를 `string` 타입으로 간주합니다.<br />
변수를 선언하거나 초기화시 타입이 추론되는것을 알 수 있습니다.
<br/>

![dog](/images/posts/Typescript/type-assertion/dog.png)

재할당이 가능한 let과는 달리 const는 재할당이 불가능하므로 string이 아닌 <br/>
더 타입을 좁혀 타입을 `웰시코기`로 추론한것을 확인 할 수 있습니다.
<br/>

![func](/images/posts/Typescript/type-assertion/func.png)

매개변수로 숫자를 받고 5를 넘는다면 string를 그 이하라면 number를 리턴하는 함수가 있습니다. <br/>
타입스크립트는 이 처럼 조건에 따라 다른 타입을 리턴하게 되더라도 그 맥락을 읽고 <br/>
함수가 string 혹은 number 타입을 리턴하는 것을 추론합니다.

## 타입 단언

때론 개발자가 컴파일러보다 타입에 대해 더 많은 정보를 갖고 있을수 있습니다. <br/>
타입 단언은 컴파일러에게 개발자가 타입을 확신 할 수 있도록 명시적으로 정의하는 기능입니다. <br/>
<br/>

![cat](/images/posts/Typescript/type-assertion/cat.png)

위처럼 값이 let에 할당되었지만 as const로 타입 단언을 하니 <br/>
타입이 랙돌로 추론된 것을 확인할 수 있습니다.

## 활용 1

![gear1](/images/posts/Typescript/type-assertion/gear1.png)

Object타입의 데이터는 원시 값처럼 변수에 직접 데이터가 저장되는 방식이 아닌 <br/>
참조 주소가 할당되기 때문에 const에 할당한다고해서 내부 값이 불변하는것은 아닙니다. <br/>
그렇기 때문에 타입스크립트도 프로퍼티를 string으로 추론하고있습니다.
<br/>

![gear2](/images/posts/Typescript/type-assertion/gear2.png)

이때 타입 단언을 이용하면 할당된 값으로 타입을 좁힐수 있고 <br/>
객체를 상수로써 활용할수 있고 혹은 enum과 같은 방법으로 활용할수 있습니다.

## 활용 2

![gear3](/images/posts/Typescript/type-assertion/gear3.png)

코드의 흐름 상 객체에 프로퍼티가 할당 될 것으로 기대되지만 <br/>
객체가 정의된 시점에선 프로퍼티가 정의 되지 않았기 때문에 다음과 같이 타입 에러가 발생하게됩니다.

![gear4](/images/posts/Typescript/type-assertion/gear4.png)

객체를 정의할때 프로퍼티를 정의하는것도 방법이지만 <br/>
위와 같이 타입 단언을 통해서도 타입 에러를 해결할 수 있습니다.

<br/>

이 처럼 개발자가 컴파일러보다 타입에 대해 더 잘 알고있는 상황이라면 <br/>
직접 타입 추론을 보완하는 방법을 기대해 볼수 있습니다.
